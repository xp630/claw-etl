## 三、组件关系与层级

### 3.1 父子关系表示

组件通过 `parentId` 字段建立层级关系：

```typescript
// 根组件（无 parentId）
{ id: "comp_1", type: "text", label: "标题", parentId: undefined }

// 子组件（有 parentId）
{ id: "comp_2", type: "input", label: "输入框", parentId: "comp_1" }
```

**树形结构示例**：

```
根组件们（无 parentId）
├── 组件A
│   └── 子组件A1（parentId: "A_id"）
│   └── 子组件A2（parentId: "A_id"）
├── 组件B
│   └── 子组件B1（parentId: "B_id"）
```

### 3.2 同级组件顺序

多个组件拥有相同 `parentId` 时，通过**数组索引**决定顺序：

| 位置 | 存储方式 | 顺序决定方式 |
|------|----------|--------------|
| 根组件 | `components` 数组 | 数组索引 |
| 容器子组件 | `children` 数组 | 数组索引 |
| Tabs 子组件 | `childrenMap[tabIndex]` 数组 | 数组索引 |

**根组件顺序示例**：

```typescript
// PageEditor.tsx
const [components, setComponents] = useState<CanvasComponent[]>([]);

// components[0] = 第一个组件（最上）
// components[1] = 第二个组件
// components[2] = 第三个组件（最下）
// 顺序就是数组索引顺序
```

### 3.3 位置处理机制

**当前实现：Flex 相对布局（不支持绝对定位）**

```tsx
// DropCanvas.tsx - 使用 Flexbox 布局
<div className="flex flex-col gap-3">
  {components.map((comp, index) => 
    renderComponentItem(comp, index, null, false)
  )}
</div>
```

| 特性 | 状态 | 实现方式 |
|------|------|----------|
| 相对位置 | ✅ 支持 | Flexbox（flex-col） + gap |
| 绝对定位 | ❌ 不支持 | 画布使用 Flex 相对布局 |
| 调整顺序 | ✅ 支持 | 上移/下移按钮 |

## 四、容器组件特殊处理

### 4.1 容器组件判断

```typescript
const isContainerType = (type: string) => 
  type === 'card' || type === 'tabs' || type === 'collapse';
```

### 4.2 卡片（card）

**特点**：直接通过 `children` 数组管理子组件

```typescript
// 数据结构
{
  id: "card_1",
  type: "card",
  props: {
    title: "卡片标题",
    bordered: true
  },
  children: [comp_A, comp_B, comp_C]  // 直接从 children 获取顺序
}
```

### 4.3 标签页（tabs）

**特点**：使用 `childrenMap` 按标签页存储子组件 ID

```typescript
// 数据结构
{
  id: "tabs_1",
  type: "tabs",
  props: {
    tabs: ["标签1", "标签2", "标签3"],
    activeTab: 0,
    childrenMap: {
      "0": ["comp_A", "comp_B"],  // 标签1的子组件（顺序由数组决定）
      "1": ["comp_C"],              // 标签2的子组件
      "2": []                       // 标签3（空）
    }
  },
  children: [comp_A, comp_B, comp_C]  // 所有子组件扁平存储
}
```

**渲染逻辑**：

```typescript
// ComponentRenderer.tsx
if (type === 'tabs') {
  const tabIndex = String(props.activeTab || 0);
  const childrenMap = props.childrenMap as Record<string, string[]>;
  
  // 从 childrenMap 获取当前tab的子组件ID列表
  const childIds = childrenMap?.[tabIndex] || [];
  
  // 从 allComponents 中查找对应的组件
  const childComponents = childIds
    .map(id => findById(id))
    .filter(Boolean);
  
  // 渲染
  return (
    <Tabs activeKey={tabIndex}>
      {props.tabs.map((tab, i) => (
        <Tabs.Tab key={String(i)} tab={tab}>
          {childComponents.map(child => (
            <ComponentRenderer key={child.id} component={child} />
          ))}
        </Tabs.Tab>
      ))}
    </Tabs>
  );
}
```

### 4.4 折叠面板（collapse）

**特点**：通过 `childrenMap` 按面板索引存储

```typescript
// 数据结构
{
  id: "collapse_1",
  type: "collapse",
  props: {
    panels: [
      { title: "面板1", content: "" },
      { title: "面板2", content: "" }
    ],
    accordion: false
  },
  childrenMap: {
    "0": ["comp_A"],  // 面板1的子组件
    "1": ["comp_B"]   // 面板2的子组件
  },
  children: [comp_A, comp_B]
}
```

## 五、拖拽逻辑

### 5.1 拖拽类型

| 拖拽类型 | 描述 | 源位置 | 目标位置 |
|----------|------|--------|----------|
| **新增组件** | 从面板拖入画布 | ComponentPanel | DropCanvas |
| **排序** | 在画布内调整顺序 | DropCanvas (根级) | DropCanvas (根级) |
| **嵌套** | 移入/移出容器 | 画布内任意位置 | 容器组件内 |

### 5.2 拖拽状态管理

```typescript
const dragState = useRef({
  dragType: 'new' | 'reorder' | 'nested',  // 拖拽类型
  sourceId: string | null,                  // 被拖拽组件ID
  sourceContainerId: string | null,         // 源容器ID（嵌套拖拽时）
  sourceIndex: number,                      // 源索引
});
```

### 5.3 新增组件拖拽流程

```
ComponentPanel 拖拽开始
       │
       ▼
设置 dragType = 'new'
设置 dragData = { type, label, fromPalette: true }
       │
       ▼
DropCanvas onDragOver（显示放置区域）
       │
       ▼
DropCanvas onDrop
       │
       ▼
根据放置位置判断：
├── 落在根级 → 添加到 components 数组
└── 落在容器内 → 调用 onAddChildToContainer
```

### 5.4 排序拖拽流程

```
根组件拖拽开始
       │
       ▼
设置 dragType = 'reorder'
设置 sourceIndex = 当前索引
       │
       ▼
拖动到新位置
       │
       ▼
onReorder(fromIndex, toIndex)
       │
       ▼
更新 components 数组顺序
```

### 5.5 嵌套拖拽流程

```
组件拖入容器
       │
       ▼
设置 dragType = 'nested'
设置 sourceContainerId = 原容器ID
       │
       ▼
容器 onDrop
       │
       ▼
调用 handleAddChildToContainer
       │
       ▼
更新容器的 childrenMap
```

## 六、保存与加载机制

### 6.1 保存时机

| 时机 | 触发条件 | 说明 |
|------|----------|------|
| 自动保存 | 新页面添加第一个组件时 | 避免数据丢失 |
| 自动保存 | 拖拽排序完成时 | 实时保存布局 |
| 手动保存 | 点击保存按钮 | 用户主动触发 |

### 6.2 保存数据结构

```typescript
// savePageConfig 发送的数据结构
interface PageConfigData {
  id?: number;                    // 页面ID（更新时需要）
  name: string;                   // 页面名称
  code?: string;                  // 页面代码
  status?: number;               // 状态
  components: CanvasComponent[];  // 组件树（完整）
}
```

### 6.3 加载流程

```typescript
// 1. 获取页面详情
const data = await getPageConfig(pageId);

// 2. 扁平数据
const flatComponents = data.components;  // 带有 parentId 的扁平数组

// 3. 重建树结构
const loaded = buildComponentTree(flatComponents);

// 4. 设置到状态
setComponents(loaded);
```

### 6.4 树结构重建算法

```typescript
const buildComponentTree = (flatComponents: Record<string, any>[]): CanvasComponent[] => {
  // 1. 创建所有组件引用
  const compMap = new Map<string | number, CanvasComponent>();
  flatComponents.forEach(c => {
    compMap.set(c.id, { ...c, children: [] });
  });

  // 2. 构建父子关系
  flatComponents.forEach(c => {
    const comp = compMap.get(c.id);
    const parentId = c.parentId;
    
    if (parentId) {
      const parent = compMap.get(parentId);
      if (parent) {
        parent.children = parent.children || [];
        parent.children.push(comp);
      }
    }
  });

  // 3. 返回根级组件（无 parentId）
  return flatComponents
    .filter(c => !c.parentId)
    .map(c => compMap.get(c.id));
};
```

## 九、总结

### 9.1 核心设计要点

| 要点 | 说明 |
|------|------|
| 数据结构 | 树形结构（parentId + children） |
| 位置管理 | Flex 相对布局，数组索引决定顺序 |
| 容器特殊处理 | Tabs 使用 childrenMap 按标签存储 |
| 拖拽三种类型 | 新增、排序、嵌套 |
| 保存机制 | 自动保存 + 手动保存，扁平化传输 |

### 9.2 当前限制

| 限制 | 说明 |
|------|------|
| 绝对定位 | 暂不支持（使用 Flex 布局） |
| 网格布局 | 暂不支持（使用 Flex 布局） |
| 响应式断点 | 暂不支持 |
