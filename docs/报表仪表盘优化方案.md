## 一、问题分析

### 1.1 页面打不开（操作后无响应）

**根因定位**：

| 问题点 | 位置 | 说明 |
|--------|------|------|
| API路径问题 | `reportApi.ts` | `baseURL: '/etl-admin/report'` 使用相对路径，如果前端部署在子路径下，API请求全部404 |
| 空状态处理 | `Dashboard.tsx` L155-165 | 当 `currentDashboard` 为 null 时只显示 Empty 组件，用户不知道发生了什么 |
| API错误被吞掉 | `useDashboard.ts` | `getDashboard` 失败时只设置 error 字符串，不展示给用户 |
| 图表数据流断裂 | `ChartShell.tsx` | `chart` prop 被传入但从未使用，图表组件无法渲染数据 |

**关键问题**：图表数据从未被传入！

```tsx
// Dashboard.tsx 中 ChartShell 的调用
<ChartShell chart={chart} filters={getActiveFilters()} />

// 但 ChartShell 只使用 children，从不看 chart prop！
export const ChartShell: React.FC<ChartShellProps> = ({
  data,      // 被忽略
  loading,   // 使用
  error,     // 使用
  children,  // 使用
  chart,     // 被传入但完全忽略！
}) => { ... }
```

### 1.2 配置流程繁琐

- 仪表盘 → 数据集 → 字段 至少3步配置才能看到图表
- 没有预览功能
- 图表类型只能通过"添加图表"弹窗选择，不能直观看到效果

## 二、静态展示版本方案

提供一个**无需配置即可查看效果**的静态展示模式，让用户：
1. 打开页面立即看到预设的示例图表
2. 了解仪表盘的功能和效果
3. 作为模板快速创建自己的仪表盘

### 3.2 增加错误边界

新增 `ErrorBoundary.tsx`：

```tsx
import React from 'react';
import { Result, Button } from 'antd';

export class ChartErrorBoundary extends React.Component<
  { children: React.ReactNode; fallback?: React.ReactNode },
  { hasError: boolean; error: Error | null }
> {
  state = { hasError: false, error: null };

  static getDerivedStateFromError(error: Error) {
    return { hasError: true, error };
  }

  render() {
    if (this.state.hasError) {
      return this.props.fallback || (
        <Result
          status="error"
          title="图表渲染失败"
          subTitle={this.state.error?.message}
          extra={<Button onClick={() => window.location.reload()}>刷新页面</Button>}
        />
      );
    }
    return this.props.children;
  }
}
```
